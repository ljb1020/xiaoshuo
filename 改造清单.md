
# 第一梯队：最值，先做这些

## 1) `scripts/linter.py`

### 要改什么

把 `load_rules()` 里的硬编码规则彻底拆出去，不再把规避词和 AI 味词表写死在 Python 里。现在这些规则还都在代码里

让 `linter.py` 只负责：

* 输出结果

不要再负责保存具体词表。

### 推荐落地

* `--config <path>`
* 或自动读取 `$NOVEL/config/linter.yaml`

### 为什么最值

## 2) `新增：$NOVEL/config/linter.yaml`

### 要加什么

每本小说自己的 lint 规则配置文件。

### 建议结构

```yaml
censorship_rules:
  - pattern: "(?<!龙)中国(?!.*龙国)"
    replacement: "龙国"
  - pattern: "(?<![樱花])日本"
    replacement: "樱花国"

ai_warning_patterns:
  - pattern: "总而言之"
    suggestion: "机械过渡词，建议改写"
  - pattern: "倒吸一?口凉气"
    suggestion: "机械反应词，建议用具体动作替代"

ignore:
  skip_frontmatter: true
  skip_code_fence: true
  skip_blockquote: true
  skip_tables: false
```

### 为什么要单独放小说目录

因为不同小说的避讳词、禁词、油腻词、可接受表达都不一样。

# 第二梯队：平台能力升级

## 6) `scripts/compress_timeline.py`

### 要改什么

现在还是“保留最近 20 条”的机械压缩 
下一步别只做“裁剪”，要做“分层”。

### 推荐拆法

不要只维护一个 `timeline.md`，改成：

* `plot/timeline_recent.md`
* `plot/timeline_canonical.md`
* `plot/timeline_resolved.md`
* `plot/timeline_archive/`

### `compress_timeline.py` 改造目标

把它从“截断脚本”改成“记忆整理器”：

* 早期但仍长期有效的事实 → canonical
* 已完成闭环的阶段事件 → resolved
* 最近章节局部上下文 → recent
* 纯原始存档 → archive

### 为什么很重要

长篇写作最大的问题不是写不出来，是记忆慢性崩坏。
